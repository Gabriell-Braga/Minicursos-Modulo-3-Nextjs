Módulo 2 · Next.js

Rotas & layouts

FILE-BASED ROUTING, LINK E PÁGINAS ANINHADAS

Objetivos

Entender o roteamento por arquivos e estruturar layouts reutilizáveis.

Agenda do módulo

  • File-based routing no App Router.
  • Pastas aninhadas e páginas.
  • layout.tsx envolvendo children.
  • Diferença entre Link e <a>.
  • Metadados por página, route groups e 404.

File-based routing

No Next.js, cada pasta dentro de app/ representa um segmento de URL. A presença de page.tsx cria uma rota acessível.

Mental model:

app/sobre/page.tsx → /sobre

Estrutura mínima

app/
app/
  layout.tsx
  page.tsx
  sobre/
    page.tsx
  contato/
    page.tsx

Pastas aninhadas

Subpastas criam segmentos encadeados:

app/blog/post/page.tsx → /blog/post

Isso torna a estrutura de arquivos previsível para toda a equipa.

Rotas dinâmicas (visão rápida)

Nomes entre colchetes capturam parâmetros de URL.

app/produtos/[id]/page.tsx

A rota /produtos/42 injeta params.id = "42".

layout.tsx e children

Layouts envolvem as páginas do segmento e evitam repetição de estrutura comum.

export default function RootLayout({ children }: {
  children: React.ReactNode
}) {
  return (
    <html lang="pt-BR">
      <body>
        <Header />
        {children}
      </body>
    </html>
  );
}

Root layout vs layout de segmento

  • Root layout: obrigatório em app/layout.tsx.
  • Layouts internos: opcionais em subpastas para secções específicas.
  • Composição: um layout interno é renderizado dentro do root.

Link vs <a>

next/link <a> tradicional
Navegação de app sem recarregar página inteiraPode causar reload completo
Prefetch automático em várias situaçõesSem prefetch embutido
Ideal para rotas internasÚtil para links externos e downloads

Metadata por página

Cada rota pode exportar metadados para título e descrição da aba.

export const metadata = {

title: 'Contato',

description: 'Fale com nossa equipa'

};

Route groups (breve)

Pastas com parênteses organizam arquivos sem alterar a URL pública.

app/(marketing)/sobre/page.tsx continua em /sobre.

Use para separar áreas (marketing, dashboard, auth) sem poluir o caminho.

Conceito de 404

  • Rotas inexistentes mostram página de não encontrado.
  • Você pode personalizar com app/not-found.tsx.
  • Ajuda usuário e SEO com feedback claro de URL inválida.

Boas práticas de rotas

  • Nomeie pastas com semântica de negócio, não de componente.
  • Mantenha o root layout enxuto e reutilizável.
  • Centralize links de navegação para evitar divergência.
  • Pense em metadados desde o início do módulo.

Checklist de teoria

Rota nasce de page.tsx.
layout.tsx recebe children.
Use Link em navegação interna.
Metadata por rota melhora contexto e SEO.

Passo 5/20 · Trocar âncoras por Link

Substitua <a href> por <Link> — navegação client-side sem recarregar a página inteira.

src/components/Header.tsx
import Link from "next/link";
import { navLinks } from "@/lib/site";

{/* Dentro do <nav>: */}
{navLinks.map((link) => (
  <Link
    key={link.href}
    href={link.href}
    className="text-sm font-medium text-slate-400 hover:text-white"
  >
    {link.label}
  </Link>
))}

{/* ❌ Evite para rotas internas: */}
<a href="/precos">Preços</a>
{/* ✅ Prefira: */}
<Link href="/precos">Preços</Link>

Passo 15/20 · PricingFaq accordion

Accordion client — um item aberto por vez com openIndex.

src/components/PricingFaq.tsx
"use client";
import { useState } from "react";
import { pricingFaqs } from "@/lib/pricing";

export function PricingFaq() {
  const [openIndex, setOpenIndex] = useState<number | null>(0);
  return pricingFaqs.map((faq, i) => (
    <button key={faq.question} onClick={() => setOpenIndex(openIndex === i ? null : i)}>
      {faq.question}
      {openIndex === i ? <p>{faq.answer}</p> : null}
    </button>
  ));
}

Passo 19/20 · ContactForm · completo

Formulário controlado com loading simulado e tela de sucesso.

src/components/ContactForm.tsx (parte 1)
"use client";
import { useState } from "react";
import { subjectOptions } from "@/lib/contact";

export function ContactForm() {
  const [form, setForm] = useState({ nome: "", email: "", assunto: "geral", mensagem: "" });
  const [submitted, setSubmitted] = useState(false);
  const [loading, setLoading] = useState(false);

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    setLoading(true);
    await new Promise((r) => setTimeout(r, 800));
    setLoading(false);
    setSubmitted(true);
  }

  if (submitted) return <div>Mensagem enviada! Obrigado, {form.nome}.</div>;
  {/* campos no próximo passo */}
}

Campos do formulário:

<form onSubmit={handleSubmit} className="space-y-5">
  <input name="nome" value={form.nome} onChange={(e) => setForm({...form, nome: e.target.value})} required />
  <input name="email" type="email" value={form.email} onChange={...} required />
  <select name="assunto" value={form.assunto} onChange={...}>
    {subjectOptions.map((o) => <option key={o.value} value={o.value}>{o.label}</option>)}
  </select>
  <textarea name="mensagem" rows={5} value={form.mensagem} onChange={...} required />
  <button type="submit" disabled={loading}>{loading ? "Enviando…" : "Enviar mensagem"}</button>
</form>
