Módulo 3 · Next.js

Componentes & client

'USE CLIENT', HOOKS E COMPOSIÇÃO DE UI

Foco

Separar UI em componentes e decidir quando usar Client Components.

Agenda do módulo

  • Server Components vs Client Components.
  • Diretiva 'use client'.
  • Regras para hooks e interatividade.
  • Props tipadas e composição de blocos.
  • Extração para pasta components/.

Server Component por padrão

No App Router, componentes são Server Components por padrão. Eles renderizam no servidor e enviam HTML pronto.

Vantagens: menos JavaScript no cliente e melhor desempenho inicial.

Client Component quando necessário

  • Use para interações de navegador (menu, modal, tabs).
  • Necessário para hooks como useState e useEffect.
  • Necessário para APIs do browser (ex.: window, localStorage).

Diretiva 'use client'

Coloque a diretiva na primeira linha do arquivo para marcar o componente como cliente.

'use client'; import { useState } from 'react';

A diretiva afeta o arquivo e os imports que dependem dele.

Quando usar cada tipo

Server Client
Markup estático e conteúdo inicialEstados, eventos e animação interativa
Busca de dados no servidorLeitura de APIs do browser
Menos JS enviadoMais controle de UX em tempo real

Hooks só em Client

Se um componente usa useState, ele precisa ser Client Component.

Regra prática: interatividade local = 'use client'.

Props com tipagem

Defina interfaces para documentar entradas e reduzir erro em tempo de desenvolvimento.

type FeatureCardProps = { title: string; description: string; icon: string; };

Composição de componentes

Monte páginas combinando blocos pequenos e focados em uma responsabilidade.

  • Header cuida da navegação.
  • Hero destaca proposta principal.
  • Cards listam benefícios ou funcionalidades.

Pasta components/

Mover componentes para components/ facilita reuso e manutenção.

components/ Header.tsx Hero.tsx LogoStrip.tsx FeatureCard.tsx

Organização por domínio

  • Evite componentes gigantes com múltiplas funções.
  • Extraia blocos quando o markup começar a repetir.
  • Nomeie arquivos pelo papel visual/funcional.

Composição na page.tsx

A página principal vira um orquestrador de componentes.

<main> <Hero /> <LogoStrip /> <section>...cards...</section> </main>

Checklist de teoria

Server por padrão no App Router.
'use client' para hooks/eventos.
Props tipadas para contratos claros.
Composição para reuso e legibilidade.

Passo 2/20 · Header · menu mobile

Botão hambúrguer visível só em telas pequenas (md:hidden).

src/components/Header.tsx (parte 2)
<div className="relative md:hidden">
  <button type="button" aria-expanded={open}
    onClick={() => setOpen((v) => !v)}>
    <i className={`fa-solid ${open ? "fa-xmark" : "fa-bars"}`} />
  </button>
  {open ? (
    <nav className="absolute right-0 top-full z-50 rounded-xl border bg-slate-950 py-2">
      {navLinks.map((link) => (
        <Link key={link.href} href={link.href}
          onClick={() => setOpen(false)}>{link.label}</Link>
      ))}
    </nav>
  ) : null}
</div>
