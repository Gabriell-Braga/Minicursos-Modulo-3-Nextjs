Módulo 1 · Next.js

Introdução & setup

CREATE-NEXT-APP, APP ROUTER E PRIMEIRA LANDING

Continuidade

Depois de HTML e Tailwind, agora entramos no framework React focado em aplicações de produção.

Módulo 1 na prática

  • Entender o que é Next.js e quando usar.
  • Configurar um projeto com App Router e Tailwind.
  • Navegar pela estrutura inicial de pastas e arquivos.
  • Entregar a primeira tela de landing que será evoluída nos próximos módulos.

O que é Next.js?

  • Meta-framework React: adiciona convenções e ferramentas prontas para produção.
  • Roteamento por arquivos: pastas e arquivos viram URLs.
  • SSR/SSG: pode renderizar no servidor e gerar páginas estáticas.

React recap rápido

  • Componentes encapsulam UI e lógica.
  • JSX descreve interface com sintaxe parecida com HTML.
  • Props passam dados de pai para filho.
component.tsx
export default function Card(props: { titulo: string }) {
  return (
    <article className="rounded-xl border border-slate-700 p-4">
      <h3>{props.titulo}</h3>
    </article>
  );
}

Por que Next.js vs React puro?

  • Rotas automáticas por arquivos.
  • Layouts aninhados nativos.
  • Otimizações embutidas (imagem, fonte, build).
  • Renderização híbrida sem configurar tudo do zero.

App Router vs Pages Router

App Router (atual)

Baseado em pasta app/, com layouts, server components e data fetching moderno.

Pages Router (legado)

Mantido por compatibilidade, mas neste curso vamos usar apenas o App Router.

Setup

Criando a base técnica do projeto

Pré-requisitos Node.js + npm

  • Instalar Node.js LTS (já inclui o npm).
  • Confirmar versões no terminal.
terminal
node --version
npm --version

create-next-app com flags

Use o assistente já configurando TypeScript, Tailwind, App Router e pasta src.

comando base
npx create-next-app@latest meu-app \
  --ts \
  --tailwind \
  --app \
  --src-dir \
  --eslint

Estrutura inicial de pastas

tree
meu-app/
├─ src/
│  └─ app/
│     ├─ layout.tsx
│     ├─ page.tsx
│     └─ globals.css
├─ public/
└─ package.json

`layout.tsx` shell compartilhado

Define estrutura global com html e body, aplicada em todas as rotas.

src/app/layout.tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="pt-BR">
      <body className="bg-slate-950 text-slate-100 antialiased">
        {children}
      </body>
    </html>
  );
}

`page.tsx` UI da rota

Cada arquivo page.tsx representa uma página. No App Router, por padrão é Server Component.

src/app/page.tsx
export default function HomePage() {
  return <main>Olá Next.js</main>;
}

Server Components conceito

  • Rodam no servidor e enviam HTML pronto para o cliente.
  • Não usam useState ou hooks de browser por padrão.
  • Ótimos para performance inicial e acesso seguro a dados.

Tailwind no Next

Estilo global e tipografia moderna

`globals.css` com Tailwind v4

No v4, comece com @import "tailwindcss" e personalize tokens em @theme.

src/app/globals.css
@import "tailwindcss";

@theme {
  --color-brand-400: #4ade80;
  --color-brand-500: #22c55e;
  --color-brand-600: #16a34a;
}

Fonte com `next/font/google`

Carregue fontes com otimização automática e sem layout shift desnecessário.

layout.tsx
import { Urbanist } from "next/font/google";

const urbanist = Urbanist({
  subsets: ["latin"],
  variable: "--font-urbanist",
});

<body className={`${urbanist.variable} font-sans`}>

Scripts de execução dev e build

  • npm run dev: ambiente local com hot reload.
  • npm run build: build de produção otimizado.
Dica: rode o build antes de publicar para validar erros de tipagem e renderização.

`metadata` no layout

Configure título e descrição da aplicação direto no arquivo de layout.

layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "Meu App Next",
  description: "Aplicação criada no módulo de introdução.",
};

Passo 2/20 · Entender as opções do CLI

  • --ts — TypeScript em todo o projeto.
  • --tailwind — Tailwind v4 já configurado.
  • --app — App Router (pasta src/app).
  • --src-dir — código dentro de src/.
  • --eslint — lint básico desde o início.

Passo 3/20 globals.css

Aplicar tokens de marca e base escura para ficar alinhado com o curso Tailwind.

src/app/globals.css
@import "tailwindcss";

@theme inline {
  --color-brand-400: #4ade80;
  --color-brand-500: #22c55e;
  --color-brand-600: #16a34a;
  --font-sans: var(--font-urbanist), system-ui, sans-serif;
}

body {
  @apply min-h-screen bg-slate-950 font-sans text-slate-100 antialiased;
}

Passo 4/20 · Entrar na pasta e instalar

terminal
cd fluxolab-app
npm install

O npm install baixa Next.js, React e Tailwind para a pasta node_modules.

Passo 5/20 layout · fonte e metadata

Imports, Urbanist e metadata — ainda sem o JSX do layout.

src/app/layout.tsx (parte 1)
import type { Metadata } from "next";
import { Urbanist } from "next/font/google";
import "./globals.css";

const urbanist = Urbanist({
  subsets: ["latin"],
  weight: ["400", "500", "600", "700", "800"],
  variable: "--font-urbanist",
});

export const metadata: Metadata = {
  title: "FluxoLab",
  description: "Landing inicial em Next.js.",
};

Passo 7/20 layout · estrutura HTML

O componente RootLayout envolve todas as páginas do app.

export default function RootLayout({ children }: { children: React.ReactNode }) {

Passo 9/20 · page.tsx vs layout

  • layout.tsx — shell fixo (fonte, metadata, elementos globais).
  • page.tsx — conteúdo que muda por rota; na home é a landing.
  • O layout recebe {children} e injeta o conteúdo da page.

Passo 10/20 rodar o projeto

  • Entrar na pasta do app.

Passo 11/20 · Font Awesome no layout

Ícones usados no Header, cards e formulários — carregue o CDN no layout.tsx antes do Módulo 3.

src/app/layout.tsx · dentro de <html>
<html lang="pt-br" className={urbanist.variable}>
  <head>
    <link
      rel="stylesheet"
      href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css"
    />
  </head>
  <body>{children}</body>
</html>

Passo 12/20 · Hot reload

Com npm run dev rodando, salve qualquer arquivo — o navegador atualiza sozinho.

  • Teste: mude o título do hero e salve page.tsx.
  • Erros de sintaxe aparecem no terminal e no overlay do navegador.

Passo 13/20 · Validar no navegador

Abra http://localhost:3000 e confira se a home inicial aparece com tema escuro e tipografia Urbanist.

  • Título e parágrafo do hero simples visíveis.
  • Botão verde de CTA renderizando.
  • Sem erros no terminal onde roda npm run dev.

O hero inline será substituído por um componente no Módulo 3 — isso é esperado.

Passo 14/20 · Alias @/

O template já mapeia @/ para src/ — usaremos em imports como @/components/Header.

// tsconfig.json (trecho)
"paths": { "@/*": ["./src/*"] }

Passo 15/20 · Scripts do package.json

  • npm run dev — servidor local com hot reload.
  • npm run build — build de produção (usado no Módulo 4).
  • npm run lint — verifica problemas de código.

Passo 16/20 · Erros comuns no setup

  • Porta 3000 ocupada — feche outro npm run dev ou use outra porta.
  • Pasta errada — rode comandos dentro de fluxolab-app.
  • Typo em className — Tailwind não aplica classe com nome errado.
